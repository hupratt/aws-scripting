How to setup single Jenkins with multiple different projects linked with their distinct GitHub repos

Long title, huh ? Well, what can you do !

Lately I found myself in need of setting up a single Jenkins instance with more that one projects. Well, easy you'll say, after all that's what Jenkins was built for...yet, what about linking every project with a different GitHub [private] repo? That was my use case. I wanted to have a single Jenkins installation on a server, multiple different projects and each one to have it's own distinct automated pipeline . From IDE to Github to Jenkins workspace.

Right there though I was in for a rather bad surprise. Jenkins comes by default with support for only one SSH key to link with a CVS [eg Bitbucket/Github]. It needed a little bit of hacking and slashing but finally there is a way to do it .

As I want to present a full guide , I'll describe first how to setup a single project with CVS support and then ,as a second step, how to setup multiple .

Single project setup

1. Go to Jenkins url [something.com:8686 ... I use port 8686]

2. Click New item

3. Set a name [eg NewJob] and select the Freestyle project

4. Go to NewJob->configure

5. Enter the Github project name [eg https://github.com/username/repo_name]

6. In Source Code Management , select Git and enter the repository URL in SSH format [eg git@github.com:username/project_name.git] .

You should see an error: Jenkins cannot connect to the repo ... because it cannot add the public key to the known_hosts file. Solution: You must create an SSH key in the /var/lib/jenkins/.ssh directory of the server where Jenkins is [leave it be for now and keep reading].

7. Further down in the Build Triggers section tick the Trigger builds remotely (e.g., from scripts) and enter an Authentication Token [eg antikradasmikos!!amplaoumplas] . Note: It made me very happy when I discovered that I can set it to anything I want...a world of funny possibilities opened in front of me! :]

8. Also tick Build when a change is pushed to GitHub.

9. Go to Github repo -> Settings -> Webhooks and set the payload URL with the username/password/token like this.. [eg http://username:pass@server.com:8686/job/job_name/build?token=antikradasmikos!!amplaoumplas ]

10. Check the webhook . It must return a 201 response. If it does not, double-check the username, password, token, port and essentially everything that constructs the above url.

11. [ optional: may be needed to open Jenkins port [8686] in your secgroup/firewall for 192.30.252.0/22, which is the GitHub IP range ]

12. Now, if you want to setup just one project, SSH into the server and go to /var/lib/jenkins/.ssh/ . There , create an rsa key [ssh-keygen -t rsa] as the Jenkins user, copy the public part [cat key_name.pub] and go to the deploy keys setting of your repo and add it as a new key. The key appears grey. When you'll make the first push it will be activated. [Note: this step is rather quick but below I describe it in more detail]

13. Finally we must need to reset the permissions of the new keys back to default

sudo chmod 600 ~/.ssh/key_name
sudo chmod 600 ~/.ssh/key_name.pub

Otherwise you 'll have an error like this


Multiple projects setup [or How to set up deployment keys for multiple Jenkins projects]

For multiple projects, the procedure is the same as above until the creation of the SSH key in the server. There we must create a 'scheme' that will allow us to create many keys and instruct Jenkins to point each one to it's corresponding project.

1. SSH to the server, and switch to root [sudo su]

2. Create an SSH config file in the jenkins users ~/.ssh directory [nano /var/lib/jenkins/.ssh/config]

3. In this file, add an entry/alias for each project 

# plain text config file inside /var/lib/jenkins/.ssh/# Alias for project1
Host the-project1
HostName github.com
User git
IdentityFile ~/.ssh/project1_key

# Alias for project2
Host the-project2
HostName github.com
User git
IdentityFile ~/.ssh/project2_key

4. Switch to Jenkins user to create the deploy keys [sudo su jenkins]

5. Create deployment keys for the different projects [but don't use passphrase] . Ensure that your keys are owned by the jenkins user. [ssh-keygen -t rsa]

6. Add the public part of each key [cat deploy_key.pub] to the deploy keys section of their associated repo.

7. Optional: If there is no known_hosts file inside /var/lib/jenkins/.ssh do the following

sudo su jenkins
ssh git@the-project1

It forces the creation of the file.

8. Set up Jenkins to pull from the repos. Go to the configuration page of each project/job, and at the repo URL input field change this 

git@github.com:username/repo_name.git

to this

git@the-project1:username/repo_name.git

Note: in the above picture I assume the repo name is the same with the project name.

9. Well done. You are set and ready. Push some code from your IDE in both repos and watch the builds take place.
